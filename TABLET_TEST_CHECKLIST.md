# 태블릿 테스트 체크리스트 (2026-06-16)

이번(2026-06-15) 작업분 검증용. 태블릿은 폰과 **다른 레이아웃 경로**를 쓰므로(앱: 너비≥600 → NavigationRail / 위젯: `tablet_widget_layout`) 각 항목을 태블릿에서 직접 확인할 것.

## 0. 준비
- [ ] 태블릿 무선 ADB 연결 (페어링 → connect)
- [ ] 최신 빌드 설치: `puro flutter build apk --debug` 후 `adb install -r` (커밋 14cbceb 포함분)
- [ ] 로그인 정상 (Tasks/Calendar/Gmail 데이터 로드)

## 1. 위젯 즉시 반영 (커밋 6732097) — 핵심
> 원인이던 ClassNotFoundException 이 태블릿에서도 안 나는지 확인
- [ ] 홈 화면에 **태블릿 위젯** 배치
- [ ] 앱에서 **할 일 추가** → 태블릿 위젯에 **즉시** 나타남
- [ ] 앱에서 **할 일 완료 체크** → 위젯에서 즉시 사라짐/반영
- [ ] 앱에서 **할 일 스와이프 삭제** → 위젯에 즉시 반영
- [ ] 앱에서 **일정(캘린더) 추가/삭제** → 위젯 캘린더에 즉시 반영
- [ ] (선택) 새 메일 수신/읽음 → 위젯 Gmail 섹션 반영
- [ ] 태블릿 위젯 3개 섹션(태스크·캘린더·Gmail) **동시 표시** 정상 (태블릿은 분할 레이아웃)

## 2. SnackBar 자동 닫힘 (커밋 75afa66, 9ab61a0)
> 태블릿(NavigationRail 레이아웃)에서도 안내 메시지가 3초 후 사라지는지
- [ ] **할 일 삭제** → "삭제됨" 스낵바가 약 3초 후 사라짐, "취소" 누르면 복구
- [ ] **태스크 추가 실패**(비행기모드 등) → 안내 스낵바 자동 사라짐
- [ ] **일정 저장 시 제목 미입력** → 경고 스낵바 자동 사라짐
- [ ] **메일 첨부파일 다운로드 완료/실패** → 스낵바 자동 사라짐, "열기" 동작
- [ ] **일정 삭제 / 캘린더 내 할 일 삭제** → 스낵바 자동 사라짐
- [ ] **로그아웃 후 로그인 실패** → 로그인 화면 스낵바 자동 사라짐

## 3. 다국어 (하드코딩 한글 점검)
> 사용자 노출 문자열은 모두 i18n 처리됨. 태블릿 영어 환경에서 한글 잔존 없는지 눈으로 확인
- [ ] 기기 언어 **English** 로 변경 후 앱 재실행
- [ ] 앱 전 화면(태스크/캘린더/Gmail/설정) 영어 표시, 한글 잔존 없음
- [ ] **태블릿 위젯**: 월 이름(예: "June 2026"), 요일, "All day", "(No subject)" 등 영어 표시
- [ ] 알림(새 메일) 영어 표시
- [ ] 기기 언어 **한국어** 복귀 후 정상

## 4. 태블릿 고유 렌더링 (이번 수정과 무관하나 함께 점검)
- [ ] 태블릿 위젯 캘린더 셀 일정 슬롯(최대 12개) 표시/오버플로 정상
- [ ] 날짜 탭 → 일 상세 패널 전환 정상
- [ ] 위젯 크기 조절 시 글씨 크기 반응형 조정 정상
- [ ] 위젯 ⚙ 설정 → WidgetConfigureActivity 정상

## 메모
- 폰(Z Fold4)에서는 1·2번 모두 검증 완료. 태블릿은 레이아웃 경로가 달라 재확인하는 것.
- 이상 발견 시: `adb logcat -s flutter:* HomeWidget:*` 로 로그 캡처 후 공유.
